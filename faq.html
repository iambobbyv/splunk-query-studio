Getting Started
What is Splunk Query Studio?

Splunk Query Studio is a desktop and mobile tool that helps SOC analysts and IT teams generate optimized Splunk SPL (Search Processing Language) queries — without memorizing syntax.

Select your data category (Network, Server, Data Center, or Applications), choose your specific platform, fill in parameters like IP addresses and time ranges, and the app builds a ready-to-use SPL query for you.

The app is fully offline — no data is ever sent to external servers.
How do I generate a query?

Follow the 4-step wizard:

  1. Pick a category — Network Operations, Server Operations, Data Center, or Applications.
  2. Select your platform — Cisco, Palo Alto, Linux, VMware, etc.
  3. Fill in parameters — IP address, time range, query type, and any field filters.
  4. Review and copy the generated SPL query.

You can jump between any completed steps by clicking the progress bar at the top.

How do I copy the generated SPL query?

On Step 4, click the Copy SPL button in the header or in the results panel. The query is placed on your clipboard, ready to paste into Splunk's search bar.

You can also use the keyboard shortcut Ctrl+Shift+C (Windows/Linux) or ⌘+Shift+C (Mac) to copy from any step.

How do I start a new query?

On Step 4, click New Query to reset all parameters and return to Step 1. You can also use Ctrl+Shift+N (or ⌘+Shift+N) from any step.

AI Assistant
Ollama is not detected — AI features aren't working

The AI chat requires Ollama running locally at http://localhost:11434. Follow these steps:

  1. Install Ollama from ollama.com
  2. Pull a supported model: ollama pull qwen2.5-coder
  3. Start the server: ollama serve
  4. In the app, click Retry in the AI status bar at the top of the AI Advisor panel.
Even without Ollama, the SPL knowledge base — command lookup, templates, and optimization hints — works completely offline with no setup required.
Which AI models are supported?

Any model installed in Ollama is supported. The app automatically selects the best available model for SPL reasoning. Recommended models (smallest first):

  • qwen2.5-coder — 1.9 GB — best code/SPL reasoning at this size
  • phi3.5 — 2.2 GB — strong reasoning, 128K context
  • llama3.2 — 2.0 GB — solid general purpose
  • gemma3 — 3.3 GB — Google, strong reasoning
  • mistral — 4.1 GB — reliable fallback
Does the AI connect to the internet?

No. The AI assistant communicates only with Ollama running on localhost:11434 — your own machine. No queries, IP addresses, or conversation data are sent externally. The static knowledge base is bundled with the app and requires no network connection.

Queries & Presets
The generated query looks wrong or incomplete

Generated queries are optimized templates for common use cases. You may need to adjust the following to match your Splunk environment:

  • Index name — replace index=* with your actual index (e.g. index=network_logs)
  • Sourcetype — match your Splunk sourcetype configuration
  • Field names — field names vary by data source and parsing rules

Use the AI Advisor panel to refine the query, or ask it to explain specific SPL commands.

How do I save and reload presets?

On Step 4, click Save Preset in the header or the results panel. Enter a name — the entire wizard state (category, platform, parameters, and query) is saved locally.

Access your presets in the right-hand panel under the Presets tab. Click any preset to restore it instantly. Presets are stored in your browser's localStorage and persist between app sessions.

What do the optimization warnings mean?

The AI Advisor panel analyzes your query against known SPL best practices and flags potential performance issues, such as:

  • No time bound — missing earliest= / latest= causes full-index scans
  • Wildcard index — index=* searches all indexes, which is slow
  • Leading wildcard — =*value cannot use the TSIDX index
  • Transaction overuse — | transaction is memory-intensive; consider | stats instead

These are warnings, not errors. The generated SPL is still valid — optimizations are recommendations to improve performance in production.

Platform & Compatibility
Which Splunk versions are supported?

Generated SPL is compatible with Splunk Enterprise 7.x and later, and Splunk Cloud. The app does not connect to your Splunk instance — queries are generated locally and you paste them into Splunk's search bar.

Does it work on iPhone, iPad, or mobile?

Yes. Splunk Query Studio is fully responsive and optimized for iPhone and iPad. On mobile, the wizard uses a full-screen slide layout for easy step-by-step navigation.

  • Tap the ☰ hamburger icon to open the category sidebar
  • Tap the clock icon to open the History and AI Advisor panel
  • All touch targets are sized for comfortable one-handed use
Which desktop operating systems are supported?

The desktop app supports macOS (Apple Silicon and Intel) and Windows (10/11). It is built with Electron and runs natively on both platforms.

Does the app send any data to the internet?

No. Splunk Query Studio is fully offline. No queries, IP addresses, or usage data are ever transmitted. All data including query history and saved presets is stored locally on your device.

The AI assistant only communicates with Ollama on localhost. External links open in your default browser, but no data is sent automatically.
